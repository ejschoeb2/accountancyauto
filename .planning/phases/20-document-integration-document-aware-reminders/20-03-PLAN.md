**Export 2: `resolveDocumentsRequired`**

```typescript
export async function resolveDocumentsRequired(
  supabase: SupabaseClient,
  clientId: string,
  filingTypeId: string
): Promise<string>
```

Implementation:
1. Call `resolveEffectiveChecklist(supabase, clientId, filingTypeId)`.
2. Filter to `mandatoryItems` where `is_mandatory = true`.
3. If `mandatoryItems.length === 0`, return `''`.
4. Fetch `client_documents` for `clientId + filingTypeId` with `classification_confidence` IN `('high', 'medium')`. Select `document_type_id` only.
5. Build `receivedSet: Set<string>` of satisfied `document_type_id` values.
6. `outstanding = mandatoryItems.filter(item => !receivedSet.has(item.documentTypeId))`.
7. If `outstanding.length === 0`, return `''` (all received).
8. Build HTML: `<ul>${outstanding.map(item => '<li>' + item.label + '</li>').join('')}</ul>`.
9. Return the HTML string.

The HTML format is per the locked decision in CONTEXT.md: `<ul><li>...</li></ul>`. This is system-generated from seeded HMRC document type labels â€” not user-controlled input. Safe for injection into the rendered email body via the existing `dangerouslySetInnerHTML` path (per RESEARCH.md anti-patterns analysis).

Use `SupabaseClient` from `@supabase/supabase-js` for the function parameter type.
