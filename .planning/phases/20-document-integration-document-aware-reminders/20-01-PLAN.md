<objective>
Consolidate document data fetching so the filing card page no longer fires four redundant parallel requests to /api/clients/{id}/documents on every page load.

Purpose: Four DocumentCard instances each call GET /api/clients/{id}/documents on mount and filter client-side — four identical round trips. This plan eliminates three of those by embedding doc_count + last_received_at in the filings response and adding a server-side filing_type_id filter so expanded cards fetch only their own documents.

Output: Two modified route files. No new routes, no migrations, no new packages.
</objective>
