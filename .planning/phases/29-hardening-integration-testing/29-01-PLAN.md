**Create `app/api/portal/[token]/upload-session/route.ts`:**

This POST endpoint is called by the browser when `file.size > 4 MB`. It authenticates the portal token (same pattern as upload/route.ts), resolves the org's storage backend, then initiates a provider-native upload session. Returns `{ sessionUrl, sessionId, provider, storagePath? }` to the browser.

Body shape expected (JSON): `{ filename: string, mimeType: string, fileSize: number, clientName: string, filingTypeId: string, taxYear: string }`

Logic:
1. Validate token (hash with SHA-256, query upload_portal_tokens, check revoked_at + expires_at) — identical pattern to upload/route.ts
2. Resolve org storage config (join or fallback queries, same as upload/route.ts)
3. Branch on `orgStorageBackend`:

**Google Drive branch (`storage_backend === 'google_drive'`):**
- Look up org credentials: `SELECT google_refresh_token_enc, google_access_token_enc, google_token_expiry FROM organisations WHERE id = orgId` (service client)
- Decrypt tokens with `decryptToken()` from `lib/crypto/tokens.ts`
- Build `GoogleCredentials` object, call `withTokenRefresh(creds, async (auth) => { ... })` from `lib/storage/token-refresh.ts`
- Inside the refresh wrapper: instantiate `createDrive({ version: 'v3', auth })`, use `findOrCreateFolder` pattern (re-implement inline — NOT importing private method; replicate the 4 queries: root → client → filingType → taxYear) to get `taxYearFolderId`
- Initiate resumable session: `POST https://www.googleapis.com/upload/drive/v3/files?uploadType=resumable` with `Authorization: Bearer {accessToken}`, headers `X-Upload-Content-Type: {mimeType}`, `X-Upload-Content-Length: {fileSize}`, body `{ name: filename, parents: [taxYearFolderId] }`
- Extract `sessionUrl = response.headers.get('Location')`
- Return `{ sessionUrl, sessionId: null, provider: 'google_drive', storagePath: null }` — the Drive file ID comes back in the final chunk response to the browser; the finalize endpoint receives it

**OneDrive branch (`storage_backend === 'onedrive'`):**
- Instantiate `OneDriveProvider(orgId, orgMsHomeAccountId)`, call its token-acquisition internally (MSAL). Alternatively: fetch `ms_token_cache_enc` from organisations, decrypt, build MSAL client with `PostgresMsalCachePlugin` to get a valid access token
- Create upload session: `POST https://graph.microsoft.com/v1.0/me/drive/root:/Apps/Prompt/{encPath}:/createUploadSession` with `Authorization: Bearer {accessToken}`, body `{ item: { '@microsoft.graph.conflictBehavior': 'rename' } }`, where `encPath = [clientName, filingTypeId, taxYear, filename].map(encodeURIComponent).join('/')`
- Extract `uploadUrl` from response JSON
- Return `{ sessionUrl: uploadUrl, sessionId: null, provider: 'onedrive', storagePath: null }`

**Dropbox branch (`storage_backend === 'dropbox'`):**
- Dropbox has no pre-authenticated session URL — use a special marker: return `{ sessionUrl: null, sessionId: 'PROXY', provider: 'dropbox', storagePath: null }` — the browser will use the streaming proxy path instead of direct chunk uploads

**Supabase branch (default):**
- Return `{ sessionUrl: null, provider: 'supabase', sessionId: null }` — browser falls back to existing upload route

Modify `portal-checklist.tsx` to route large files through the chunked path.

**Constant:** `const LARGE_FILE_THRESHOLD = 4 * 1024 * 1024; // 4 MB — below Vercel 4.5 MB limit`

**Chunk size:** `const CHUNK_SIZE = 1310720; // LCM(262144, 327680) = 1.25 MB — satisfies both Drive (256 KiB multiples) and OneDrive (320 KiB multiples)`

**New helper `uploadInChunks(sessionUrl: string, file: File, onProgress?: (pct: number) => void): Promise<{ fileId: string }>`:**
```typescript
async function uploadInChunks(sessionUrl, file, onProgress) {
  let offset = 0;
  while (offset < file.size) {
    const end = Math.min(offset + CHUNK_SIZE, file.size) - 1;
    const chunk = file.slice(offset, end + 1);
    const chunkBuffer = await chunk.arrayBuffer();
    const res = await fetch(sessionUrl, {
      method: 'PUT',
      headers: {
        'Content-Length': String(chunkBuffer.byteLength),
        'Content-Range': `bytes ${offset}-${end}/${file.size}`,
        // NOTE: NO Authorization header — session URL is pre-authenticated for Drive/OneDrive
      },
      body: chunkBuffer,
    });
    if (res.status === 200 || res.status === 201) {
      const data = await res.json();
      return { fileId: data.id };
    }
    if (res.status !== 202) throw new Error(`Chunk upload failed (${res.status})`);
    offset = end + 1;
    onProgress?.(Math.round((offset / file.size) * 100));
  }
  throw new Error('Upload loop exited without completion response');
}
```

const handleUpload = async (itemId: string, file: File, confirmDuplicate = false) => {
  // Large file path: provider-native chunked upload for non-Supabase backends
  if (file.size > LARGE_FILE_THRESHOLD) {
    // Step 1: request session from server
    const sessionRes = await fetch(`/api/portal/${rawToken}/upload-session`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        filename: file.name,
        mimeType: file.type,
        fileSize: file.size,
        clientName: '', // not needed for session init routing; server reads from token
        filingTypeId: '', // server reads from token
        taxYear: '',     // server reads from token
      }),
    });
    if (!sessionRes.ok) throw new Error('Failed to initiate upload session');
    const session = await sessionRes.json();

    if (session.provider === 'supabase') {
      // Supabase handles multipart natively — fall through to existing route below
    } else if (session.provider === 'dropbox') {
      // Dropbox: streaming proxy — send file to a proxied endpoint
      // For Dropbox large files, fall through to existing upload route with a warning
      // (Dropbox filesUpload supports up to 150 MB via the SDK on the server side;
      // the 4.5 MB body limit applies to the Vercel function request BODY, but the
      // Dropbox SDK upload call reads the buffer from server memory, not from the
      // incoming request — so large Dropbox files must still use the existing route
      // which buffers on the server. This is a known limitation noted in research.
      // For a full fix, a streaming proxy endpoint would be needed. Defer to post-Phase-29.)
      // Fall through to normal route.
    } else if (session.sessionUrl) {
      // Google Drive or OneDrive: direct chunked upload from browser to provider
      const { fileId } = await uploadInChunks(session.sessionUrl, file);

      // Step 2: finalize — write client_documents row
      const sha256Hash = await computeSha256(file); // helper below
      const finalizeRes = await fetch(`/api/portal/${rawToken}/upload-finalize`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          storagePath: fileId,
          filename: file.name,
          mimeType: file.type,
          fileSize: file.size,
          provider: session.provider,
          sha256Hash,
        }),
      });
      if (!finalizeRes.ok) {
        const d = await finalizeRes.json().catch(() => ({ error: 'Finalize failed' }));
        throw new Error(d.error ?? 'Upload finalize failed. Please try again.');
      }
      const data = await finalizeRes.json();
      setUploadedByItemId(prev => ({
        ...prev,
        [itemId]: [...(prev[itemId] ?? []), {
          filename: file.name,
          confidence: data.confidence ?? 'unclassified',
          documentTypeLabel: data.documentTypeLabel ?? null,
          extractedTaxYear: null,
          extractedEmployer: null,
          extractedPayeRef: null,
          showConfirmationCard: false, // large files skip OCR confirmation card
        }],
      }));
      return; // done
    }
  }

  // Existing small-file path (unchanged)
  const formData = new FormData();
  // ... existing code ...

**Add `computeSha256(file: File): Promise<string>` helper** using Web Crypto API:
```typescript
async function computeSha256(file: File): Promise<string> {
  const buffer = await file.arrayBuffer();
  const hashBuffer = await crypto.subtle.digest('SHA-256', buffer);
  return Array.from(new Uint8Array(hashBuffer)).map(b => b.toString(16).padStart(2, '0')).join('');
}
```

NOTE: `computeSha256` re-reads the whole file into memory — this is acceptable because at 4+ MB, the browser can handle it; the goal was preventing the SERVER from buffering, not the browser.
