<task type="auto">
  <name>Task 2: Create lib/storage/token-refresh.ts with withTokenRefresh utility</name>
  <files>lib/storage/token-refresh.ts</files>
  <action>
    Create `lib/storage/token-refresh.ts`. This is the ONLY place in the codebase that handles Google token refresh — all Drive API calls (upload, download, delete) go through this wrapper.

    The file must export:

    **`GoogleCredentials` interface:**
    ```typescript
    export interface GoogleCredentials {
      access_token_enc: string;    // AES-256-GCM encrypted access token
      refresh_token_enc: string;   // AES-256-GCM encrypted refresh token
      expires_at: string;          // ISO timestamp (google_token_expires_at column)
      org_id: string;              // Used to update organisations on refresh/error
    }
    ```

    **`withTokenRefresh<T>(creds: GoogleCredentials, call: (auth: OAuth2Client) => Promise<T>): Promise<T>`:**

    Implementation logic:
    1. Construct `OAuth2Client` lazily — use `new google.auth.OAuth2(process.env.GOOGLE_CLIENT_ID, process.env.GOOGLE_CLIENT_SECRET, process.env.GOOGLE_REDIRECT_URI)`. Follow the lazy init pattern from D-11-05-01 (Stripe client) — do NOT read env vars at module level.
    2. Check if token will expire within 5 minutes: `new Date(creds.expires_at).getTime() < Date.now() + 5 * 60 * 1000`
    3. If proactive refresh needed:
       - Decrypt refresh token: `decryptToken(creds.refresh_token_enc)`
       - Set credentials: `oauth2Client.setCredentials({ refresh_token: refreshToken })`
       - Call `oauth2Client.refreshAccessToken()`
       - On success: persist refreshed token back to organisations (encrypt new access token, update `google_token_expires_at`, set `storage_backend_status = 'active'`)
       - On `invalid_grant` error (check `err.response?.data?.error === 'invalid_grant'`): UPDATE organisations SET `storage_backend_status = 'reauth_required'`, `google_access_token_enc = null`, `google_refresh_token_enc = null`, `google_token_expires_at = null` WHERE id = creds.org_id. Then re-throw — never retry.
    4. If not expiring (or after successful refresh): decrypt access token, set `oauth2Client.setCredentials({ access_token: accessToken })`
    5. Execute `call(oauth2Client)` inside try/catch. On `invalid_grant` at call-time (e.g. access token revoked mid-flight): same DB nullification as above, then re-throw.
    6. Return result of `call`.

    Imports needed:
    ```typescript
    import { google } from '@googleapis/drive';
    import type { OAuth2Client } from 'google-auth-library';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { encryptToken, decryptToken } from '@/lib/crypto/tokens';
    ```

    Critical: Do NOT suppress invalid_grant errors. The caller (GoogleDriveProvider) must propagate them so the upload/download route returns a 500, which the caller converts to a UI error message. The reauth banner in the layout will inform the user what happened.
  </action>
