  <name>Task 1: Create app/api/auth/google-drive/connect/route.ts</name>
  <files>app/api/auth/google-drive/connect/route.ts</files>
  <action>
    Create `app/api/auth/google-drive/connect/route.ts` with a single `GET` handler.

    The route must:
    1. Verify the user is authenticated (use `createClient()` and `supabase.auth.getUser()`) — return 401 if not authenticated. Only admins should trigger this but the auth check is sufficient for now.
    2. Build an `OAuth2Client` lazily (NOT at module level — prevent build failure when env vars absent):
       ```typescript
       function getOAuth2Client() {
         return new google.auth.OAuth2(
           process.env.GOOGLE_CLIENT_ID!,
           process.env.GOOGLE_CLIENT_SECRET!,
           process.env.GOOGLE_REDIRECT_URI!,
         );
       }
       ```
    3. Generate a 32-byte hex CSRF state token: `crypto.randomBytes(32).toString('hex')`
    4. Generate the authorization URL:
       ```typescript
       const authUrl = oauth2Client.generateAuthUrl({
         access_type: 'offline',
         scope: ['https://www.googleapis.com/auth/drive.file'],
         state,
         prompt: 'consent', // CRITICAL: ensures refresh_token is always returned (not just first auth)
       });
       ```
    5. Create a `NextResponse.redirect(authUrl)` response
    6. Set the state cookie on the response:
       ```typescript
       response.cookies.set('google_oauth_state', state, {
         httpOnly: true,
         secure: process.env.NODE_ENV === 'production',
         sameSite: 'lax',
         maxAge: 600, // 10 minutes — short TTL prevents replay
         path: '/',
       });
       ```
    7. Return the redirect response

    Imports:
    ```typescript
    import { NextResponse } from 'next/server';
    import { createClient } from '@/lib/supabase/server';
    import { google } from '@googleapis/drive';
    import crypto from 'crypto';
    ```
  </action>

<task type="auto">
  <name>Task 2: Create app/api/auth/google-drive/callback/route.ts</name>
  <files>app/api/auth/google-drive/callback/route.ts</files>
  <action>
    Create `app/api/auth/google-drive/callback/route.ts` with a single `GET` handler.

    The handler must:

    1. **Auth + CSRF validation:**
       - Verify user is authenticated via `createClient()` and `supabase.auth.getUser()` — redirect to `/login` if not
       - Extract `code` and `state` from URL search params
       - Extract stored state from `request.cookies.get('google_oauth_state')?.value`
       - If state mismatch or missing: redirect to `/settings?tab=storage&error=invalid_state` (do NOT throw — redirect is cleaner UX)

    2. **Token exchange:**
       - Construct `OAuth2Client` lazily (same lazy function pattern as connect route)
       - Call `const { tokens } = await oauth2Client.getToken(code!)`
       - Guard against missing refresh_token (should not happen with `prompt=consent` but must handle gracefully): redirect to `/settings?tab=storage&error=no_refresh_token`

    3. **Create Prompt/ root folder:**
       - Set credentials: `oauth2Client.setCredentials(tokens)`
       - Create Drive client: `const driveClient = google.drive({ version: 'v3', auth: oauth2Client })`
       - Create folder:
         ```typescript
         const folderRes = await driveClient.files.create({
           requestBody: {
             name: 'Prompt',
             mimeType: 'application/vnd.google-apps.folder',
             // no parents = creates in root of user's Drive
           },
           fields: 'id',
         });
         const rootFolderId = folderRes.data.id!;
         ```

    4. **Persist encrypted tokens to organisations:**
       - Get `orgId` via `getOrgContext()`
       - Use `createAdminClient()` to UPDATE organisations:
         ```typescript
         await admin.from('organisations').update({
           storage_backend: 'google_drive',
           storage_backend_status: 'active',
           google_access_token_enc: encryptToken(tokens.access_token!),
           google_refresh_token_enc: encryptToken(tokens.refresh_token!),
           google_token_expires_at: new Date(tokens.expiry_date!).toISOString(),
           google_drive_folder_id: rootFolderId,
         }).eq('id', orgId);
         ```
       - Wrap in try/catch: on DB error, redirect to `/settings?tab=storage&error=db_error`

    5. **Clean up and redirect:**
       - Create `NextResponse.redirect('/settings?tab=storage&connected=google_drive')`
       - Delete the state cookie: `response.cookies.delete('google_oauth_state')`
       - Return the redirect

    Error handling: all errors redirect to `/settings` with an error query param rather than throwing — OAuth callbacks must never show raw error pages to users.

    Imports:
    ```typescript
    import { NextRequest, NextResponse } from 'next/server';
    import { createClient } from '@/lib/supabase/server';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { getOrgContext } from '@/lib/auth/org-context';
    import { google } from '@googleapis/drive';
    import { encryptToken } from '@/lib/crypto/tokens';
    ```
