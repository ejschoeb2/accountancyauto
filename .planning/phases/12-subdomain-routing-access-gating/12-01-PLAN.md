  <action>
Create `lib/middleware/subdomain.ts` with two exported functions:

1. `getOrgSlug(request: NextRequest): string | null` — Extracts org slug from the request:
   - If hostname includes 'localhost' or '127.0.0.1' (development): return `request.nextUrl.searchParams.get('org')`
   - If hostname matches `{slug}.app.phasetwo.uk` pattern (production): extract slug from `hostname.split('.')[0]`
   - Filter reserved subdomains: `['www', 'app', 'api', 'admin', 'billing']` — return null for these
   - Bare `app.phasetwo.uk` (no slug, parts.length < 3 or slug === 'app') → return null
   - NOTE: Only allow query param fallback in development. Check `process.env.NODE_ENV === 'development'` before allowing `?org=` param. In production, ONLY subdomain-based identification.

2. `resolveOrgFromSlug(supabase: SupabaseClient, slug: string): Promise<OrgInfo | null>` — Looks up org by slug:
   - Query `organisations` table: `select('id, slug, name, subscription_status, trial_ends_at').eq('slug', slug).single()`
   - Return null if not found or error
   - Export the `OrgInfo` type: `{ id: string; slug: string; name: string; subscription_status: string; trial_ends_at: string | null }`

Create `lib/middleware/access-gating.ts` with one exported function:

1. `enforceSubscription(request: NextRequest, org: OrgInfo): NextResponse | null` — Checks subscription status:
   - If pathname starts with `/billing`, `/auth`, `/login`, or `/api/` → return null (allow through)
   - If `subscription_status === 'active'` → return null (allow through)
   - If `subscription_status === 'trialing'` and `trial_ends_at` is in the future → return null (allow through)
   - Otherwise (cancelled, past_due, unpaid, expired trial) → return `NextResponse.redirect(new URL('/billing', request.url), 307)`
   - Import `OrgInfo` type from `./subdomain`

Update `lib/auth/org-context.ts` to add a `getCurrentOrg()` function:

1. `getCurrentOrg = cache(async (slug: string))` — Wrapped in React `cache()` for request deduplication:
   - Import `cache` from 'react'
   - Creates a Supabase client via `createClient()`
   - Queries `organisations` table: `select('id, slug, name, subscription_status, trial_ends_at, postmark_server_token, postmark_sender_domain').eq('slug', slug).single()`
   - Returns the org row or null
   - Export the return type as `OrgRow`

Keep existing `getOrgId()` and `getOrgContext()` functions unchanged.

  <name>Task 2: Rewrite middleware for subdomain routing with org validation and access gating</name>
  <files>middleware.ts, lib/supabase/middleware.ts</files>
  <action>
Rewrite `lib/supabase/middleware.ts` to integrate subdomain routing. The `updateSession` function must:

1. **Create Supabase client** with cookie handling (keep existing pattern — createServerClient with getAll/setAll).

2. **Refresh auth token** — call `supabase.auth.getUser()` (keep existing pattern).

3. **Extract org slug** — call `getOrgSlug(request)` from `lib/middleware/subdomain`.

4. **Handle no-slug cases:**
   - Public routes (`/login`, `/auth/callback`, `/auth/signout`) and API routes (`/api/`) should pass through WITHOUT requiring a slug. These routes work regardless of subdomain context.
   - If NO slug AND NOT a public/API route:
     - If user is authenticated: look up user's org via `supabase.from('user_organisations').select('org_id').eq('user_id', user.id).limit(1).single()`, then look up org slug via `supabase.from('organisations').select('slug').eq('id', orgId).single()`, then redirect to `https://${slug}.app.phasetwo.uk${pathname}` (production) or add `?org=${slug}` (development).
     - If user is NOT authenticated: redirect to marketing site `https://phasetwo.uk`
   - NOTE for dev mode: if on localhost and no `?org=` param, and user is authenticated, redirect to same URL with `?org=${slug}` appended. If not authenticated, redirect to `/login`.

5. **Handle slug present:**
   - Resolve slug to org via `resolveOrgFromSlug(supabase, slug)`.
   - If org not found (invalid slug): redirect to marketing site `https://phasetwo.uk`.
   - If user is NOT authenticated AND route is NOT public: redirect to `/login` (on the same host/subdomain so cookie stays scoped). Include `redirect=${pathname}` query param so post-login redirect works.
   - If user IS authenticated:
     - Extract `org_id` from `user.app_metadata?.org_id`.
     - If user's org_id !== org.id (wrong org subdomain): look up user's actual org slug and redirect to `https://${userOrgSlug}.app.phasetwo.uk${pathname}` (307 temporary redirect). In dev mode, redirect to `localhost:3000${pathname}?org=${userOrgSlug}`.
     - If user's org_id === org.id (correct org): run `enforceSubscription(request, org)`. If it returns a redirect, return it. Otherwise, set request header `x-org-slug` to the slug value and return `supabaseResponse`.
     - If user has no org_id in app_metadata (fallback path): query `user_organisations` to find their org. If their org matches, allow through. If not, redirect to their org's subdomain.

6. **Important cookie handling:** When creating redirect responses, copy all cookies from `supabaseResponse` to the redirect response. This preserves the refreshed auth token. Pattern:
   ```typescript
   const redirect = NextResponse.redirect(url, 307);
   supabaseResponse.cookies.getAll().forEach(cookie => {
     redirect.cookies.set(cookie.name, cookie.value);
   });
   return redirect;
   ```

Update `middleware.ts` matcher to also exclude `/api/webhooks`, `/api/cron`, `/api/postmark`, pricing page (`/pricing`), and static assets. Keep the existing exclusion pattern but ensure `/auth/callback` and `/auth/signout` are NOT excluded (they need cookie refresh but should be treated as public routes inside the middleware logic).

The matcher should be:
```
"/((?!_next/static|_next/image|favicon.ico|api/webhooks|api/cron|api/postmark|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)"
```
(This is the same as current — no change needed to matcher.)

IMPORTANT: The `x-org-slug` header should be set on the request headers that get forwarded to server components. Use this pattern:
```typescript
const requestHeaders = new Headers(request.headers);
requestHeaders.set('x-org-slug', slug);
supabaseResponse = NextResponse.next({
  request: { headers: requestHeaders },
});
// Re-set cookies on the new response
```
But be careful: this must happen AFTER the Supabase client has refreshed cookies. The cleanest approach is to create the final response at the end after all checks pass, copying cookies from the original supabaseResponse.
