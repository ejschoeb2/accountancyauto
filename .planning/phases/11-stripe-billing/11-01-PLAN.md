    1. Run `npm install stripe@^20.3.1` to add the Stripe SDK.

    2. Create a Supabase migration file with a proper timestamp (format: YYYYMMDDHHMMSS, use current date/time). The migration must:

    a) Create `processed_webhook_events` table:
       - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
       - `event_id TEXT NOT NULL UNIQUE` (Stripe event ID, e.g. `evt_xxx`)
       - `event_type TEXT NOT NULL` (e.g. `checkout.session.completed`)
       - `processed_at TIMESTAMPTZ NOT NULL DEFAULT now()`
       - Index on `event_id` for fast lookups
       - RLS enabled with service_role full access policy only (webhook handler uses admin client)
       - No authenticated access needed (webhooks bypass auth)

    b) Add `stripe_price_id TEXT` column to `organisations` table (nullable, stores the Stripe Price ID for the org's current plan):
       `ALTER TABLE organisations ADD COLUMN IF NOT EXISTS stripe_price_id TEXT;`

    c) Add index on organisations.stripe_customer_id for webhook lookups:
       `CREATE INDEX IF NOT EXISTS idx_organisations_stripe_customer_id ON organisations(stripe_customer_id);`

    Do NOT add RLS policies for `authenticated` on processed_webhook_events -- only service_role should access this table.
  </action>

<task type="auto">
  <name>Task 3: Create billing utility functions (usage limits + read-only mode)</name>
  <files>
    lib/billing/usage-limits.ts
    lib/billing/read-only-mode.ts
  </files>
  <action>
    1. Create `lib/billing/usage-limits.ts`:
       - Import createClient from '@/lib/supabase/server'
       - Export `async function checkClientLimit(orgId: string): Promise<{ allowed: boolean; currentCount: number; limit: number | null; message?: string }>`
         - Query organisations table for `client_count_limit` where id = orgId
         - If limit is null, return allowed: true (unlimited)
         - Count clients where org_id = orgId using `select('id', { count: 'exact', head: true })`
         - If count >= limit, return allowed: false with message: "You've reached your plan's client limit ({limit}). Upgrade your plan to add more clients."
         - Otherwise return allowed: true
       - Export `async function getUsageStats(orgId: string): Promise<{ clientCount: number; clientLimit: number | null; clientUsagePercent: number | null }>`
         - Query org for client_count_limit + count clients for org
         - Calculate percentage (null if unlimited)
         - Used by billing page for usage bars

    2. Create `lib/billing/read-only-mode.ts`:
       - Import createClient from '@/lib/supabase/server'
       - Export `async function isOrgReadOnly(orgId: string): Promise<boolean>`
         - Query organisations for subscription_status and trial_ends_at where id = orgId
         - Return false if subscription_status = 'active' (not read-only)
         - Return false if subscription_status = 'trialing' AND trial_ends_at > now() (valid trial)
         - Return true for all other cases: 'cancelled', 'past_due', 'unpaid', expired trial, or org not found
       - Export `async function requireWriteAccess(orgId: string): Promise<void>`
         - Calls isOrgReadOnly; if true, throws Error("Your subscription is inactive. Please update your billing to make changes.")
         - Convenience wrapper for server actions to call at the top

    Both functions use the authenticated Supabase client (server.ts) since they're called from server actions in the user's session context. RLS will automatically scope to the user's org.
  </action>

    - No imports from files that don't exist

  </done>
