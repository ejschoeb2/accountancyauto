  <name>Task 1: Create Stripe Checkout + Customer Portal API routes</name>
  <files>
    app/api/stripe/create-checkout-session/route.ts
    app/api/stripe/create-portal-session/route.ts
  </files>
  <action>
    1. Create `app/api/stripe/create-checkout-session/route.ts`:
       - Export `POST` handler
       - Authenticate user via `createClient()` + `supabase.auth.getUser()`. Return 401 if no user.
       - Parse body: `{ planTier, orgId }` from `req.json()`
       - Verify user is admin of orgId: query user_organisations for user_id + org_id, check role = 'admin'. Return 403 if not admin.
       - Import `getPlanByTier` from plans.ts -- get the plan config for the requested tier
       - If plan not found or no priceId configured, return 400
       - Check if org already has a stripe_customer_id. If yes, pass `customer: org.stripe_customer_id` to reuse existing Stripe customer. If no, pass `customer_email: user.email` to create new customer.
       - Create Stripe Checkout Session:
         ```
         stripe.checkout.sessions.create({
           mode: 'subscription',
           payment_method_collection: 'always',
           line_items: [{ price: plan.priceId, quantity: 1 }],
           success_url: `${process.env.NEXT_PUBLIC_APP_URL}/billing?session_id={CHECKOUT_SESSION_ID}`,
           cancel_url: `${process.env.NEXT_PUBLIC_APP_URL}/pricing`,
           metadata: { org_id: orgId, plan_tier: planTier },
           subscription_data: { metadata: { org_id: orgId, plan_tier: planTier } },
           customer: existingCustomerId || undefined,
           customer_email: existingCustomerId ? undefined : user.email,
           tax_id_collection: { enabled: process.env.STRIPE_TAX_ENABLED === 'true' },
           automatic_tax: { enabled: process.env.STRIPE_TAX_ENABLED === 'true' },
         })
         ```
       - Return `{ url: session.url }` as JSON

    2. Create `app/api/stripe/create-portal-session/route.ts`:
       - Export `POST` handler
       - Authenticate user via `createClient()` + `supabase.auth.getUser()`. Return 401 if no user.
       - Parse body: `{ orgId }` from `req.json()`
       - Verify user is admin of orgId (same pattern as checkout). Return 403 if not admin.
       - Fetch org from organisations where id = orgId. If no stripe_customer_id, return 400 with message "No active subscription found. Please select a plan first."
       - Create Customer Portal Session:
         ```
         stripe.billingPortal.sessions.create({
           customer: org.stripe_customer_id,
           return_url: `${process.env.NEXT_PUBLIC_APP_URL}/billing`,
         })
         ```
       - Return `{ url: session.url }` as JSON

    Both routes follow the existing pattern in the codebase: NextRequest/NextResponse, createClient() for auth, JSON body parsing.

    NOTE: Per user decision, plan changes go through Stripe Customer Portal, not in-app. So the checkout route is only for INITIAL subscription creation. Upgrades/downgrades use the portal route.
  </action>
  <verify>
    - Both route files exist and export POST
    - Both verify user authentication and admin role
    - Checkout route passes org_id in metadata for webhook handler to use
    - Portal route checks for existing stripe_customer_id
    - TypeScript compiles: `npx tsc --noEmit --pretty 2>&1 | head -20`
  </verify>
  <done>
    Checkout session creation passes org metadata to Stripe; portal session creation opens pre-authenticated Stripe Customer Portal; both enforce admin-only access
  </done>
