       - Import `requireWriteAccess` from '@/lib/billing/read-only-mode'

       - If it throws, the error propagates to the caller (existing error handling)

    3. Modify `app/actions/send-reply-email.ts`:
       - Same pattern: import requireWriteAccess, call at top of function
       - Block reply emails when subscription is lapsed

    4. Modify `app/actions/clients.ts`:
       - In `updateClientMetadata`: add requireWriteAccess(orgId) check at top (get orgId via getOrgId())
       - In `bulkUpdateClients`: add requireWriteAccess(orgId) check at top
       - In `deleteClients`: add requireWriteAccess(orgId) check at top
       - Do NOT add limit check to update/delete -- only creation needs the limit check
       - getClients (read) does NOT get a write check -- data remains visible in read-only mode per user decision

    5. Modify `app/actions/settings.ts`:
       - Add requireWriteAccess check to any setting-mutating functions (updateSendHour, updateEmailSettings, etc.)
       - Read functions (getSendHour, getEmailSettings) remain accessible in read-only mode

    Per user decision: "Lapsed subscription = read-only mode: data visible, but cannot send emails, add clients, or modify data"
