**Tables to add org_id (per ORG-04):**

**Tables to SKIP (per ORG-06 — global reference data):**
- `filing_types` — shared across all orgs
- `bank_holidays_cache` — shared across all orgs
- `oauth_tokens` — legacy, will be removed eventually
- `reminder_templates` — legacy v1.0 table (may already be dropped)
- `client_template_overrides` — legacy v1.0 table

**For each table, the pattern is:**
```sql
-- 1. Add column as nullable first
ALTER TABLE {table} ADD COLUMN org_id UUID REFERENCES organisations(id);

-- 2. Backfill all existing rows to the founding org
UPDATE {table} SET org_id = (SELECT id FROM organisations WHERE slug = 'peninsula');

-- 3. Set NOT NULL constraint
ALTER TABLE {table} ALTER COLUMN org_id SET NOT NULL;

-- 4. Add index for RLS performance
CREATE INDEX idx_{table}_org_id ON {table}(org_id);
```

**Special handling for `app_settings`:**
- app_settings currently has `key TEXT PRIMARY KEY` — no id column
- Add org_id column, backfill, set NOT NULL
- Drop the existing PRIMARY KEY constraint on `key`
- Add a new `id` UUID PK column (gen_random_uuid) if one doesn't exist
- Add UNIQUE constraint on `(org_id, key)` — this replaces the old `key`-only uniqueness (ORG-05)
- The existing `key` column stays, but is no longer the PK by itself

**Special handling for `locks`:**
- locks has `id TEXT PRIMARY KEY` — the lock key
- Add org_id, backfill, NOT NULL
- Drop old PK, add composite PK on `(id, org_id)` or add UNIQUE on `(id, org_id)` — this ensures org-scoped locks so different orgs don't block each other
- Actually, simpler: keep `id` as PK but make lock IDs org-scoped in application code (e.g., `cron_reminders_${org_id}`). Just add the org_id column with index.

**Validation at end of migration:**
```sql
-- Verify no NULL org_ids remain
DO $$
DECLARE
  tbl TEXT;
  null_count INT;
BEGIN
  FOR tbl IN
    SELECT unnest(ARRAY[
      'clients', 'email_templates', 'schedules', 'schedule_steps',
      'schedule_client_exclusions', 'client_email_overrides', 'client_schedule_overrides',
      'client_filing_assignments', 'client_deadline_overrides', 'client_filing_status_overrides',
      'reminder_queue', 'email_log', 'inbound_emails', 'app_settings', 'locks'
    ])
  LOOP
    EXECUTE format('SELECT count(*) FROM %I WHERE org_id IS NULL', tbl) INTO null_count;
    IF null_count > 0 THEN
      RAISE EXCEPTION 'Table % has % rows with NULL org_id', tbl, null_count;
    END IF;
  END LOOP;
END $$;
```

  <name>Task 3: Create validation migration with count checks</name>
  <files>supabase/migrations/20260219000003_validate_org_migration.sql</files>
  <action>
Create a validation-only migration that runs assertions to confirm the migration integrity. This is a safety net that prevents proceeding if something went wrong.

The migration should:

1. **Verify founding org exists:**
   ```sql
   DO $$ BEGIN
     IF NOT EXISTS (SELECT 1 FROM organisations WHERE slug = 'peninsula' AND plan_tier = 'firm') THEN
       RAISE EXCEPTION 'Founding org not found';
     END IF;
   END $$;
   ```

2. **Verify all FK constraints are valid** — for each table, run a check that org_id references an existing organisations row:
   ```sql
   DO $$
   DECLARE
     tbl TEXT;
     orphan_count INT;
   BEGIN
     FOR tbl IN
       SELECT unnest(ARRAY[
         'clients', 'email_templates', 'schedules', 'schedule_steps',
         'schedule_client_exclusions', 'client_email_overrides', 'client_schedule_overrides',
         'client_filing_assignments', 'client_deadline_overrides', 'client_filing_status_overrides',
         'reminder_queue', 'email_log', 'inbound_emails', 'app_settings', 'locks'
       ])
     LOOP
       EXECUTE format(
         'SELECT count(*) FROM %I t WHERE NOT EXISTS (SELECT 1 FROM organisations o WHERE o.id = t.org_id)',
         tbl
       ) INTO orphan_count;
       IF orphan_count > 0 THEN
         RAISE EXCEPTION 'Table % has % orphaned org_id rows', tbl, orphan_count;
       END IF;
     END LOOP;
   END $$;
   ```

3. **Verify user_organisations links exist:**
   ```sql
   DO $$ BEGIN
     IF (SELECT count(*) FROM user_organisations) = 0 THEN
       RAISE EXCEPTION 'No user-organisation links found';
     END IF;
   END $$;
   ```

4. **Verify app_settings unique constraint:**
   ```sql
   DO $$ BEGIN
     -- Try inserting a duplicate (org_id, key) — should fail
     -- We can't actually test this without rolling back, so just verify the constraint exists
     IF NOT EXISTS (
       SELECT 1 FROM pg_constraint
       WHERE conrelid = 'app_settings'::regclass
       AND contype = 'u'
     ) THEN
       RAISE EXCEPTION 'app_settings missing unique constraint';
     END IF;
   END $$;
   ```

5. **Log success** — insert a record into app_settings marking migration complete:
   ```sql
   INSERT INTO app_settings (org_id, key, value)
   SELECT id, 'v3_migration_complete', 'true'
   FROM organisations WHERE slug = 'peninsula'
   ON CONFLICT DO NOTHING;
   ```
