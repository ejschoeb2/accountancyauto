  <how-to-verify>
**Pre-requisite:** Enable the Custom Access Token Hook in the Supabase Dashboard:
1. Go to your Supabase project > Authentication > Hooks (under Configuration)
2. Enable "Custom Access Token (JWT)" hook
3. Set schema: `public`, function: `custom_access_token_hook`
4. Save
5. Also: Update the founding org's Postmark token: run in SQL Editor:
   ```sql
   UPDATE organisations SET postmark_server_token = '<your POSTMARK_SERVER_TOKEN>' WHERE slug = 'peninsula';
   ```

**Then verify:**

1. **JWT Claims:** Log in to the app. Open browser DevTools > Application > Cookies > find the Supabase session cookie. Decode the JWT at jwt.io. Confirm `app_metadata` contains `org_id` (matching the founding org UUID) and `org_role` = 'admin'.

2. **RLS Isolation:** Open the Supabase SQL Editor (as service_role). Run:
   ```sql
   -- As service_role, both orgs should be visible
   SELECT name, slug FROM organisations;
   -- Expected: 'Peninsula Accounting' and 'Acme Accounting (Test)'

   SELECT count(DISTINCT org_id) FROM clients;
   -- Expected: 2
   ```
   Then in the application dashboard, verify:
   - The clients list shows only Peninsula's clients (not 'Acme Test Client')
   - Settings show send hour = 9 (not 10)

3. **app_settings Constraint:** In SQL Editor:
   ```sql
   INSERT INTO app_settings (org_id, key, value)
   VALUES ((SELECT id FROM organisations WHERE slug = 'peninsula'), 'reminder_send_hour', 'duplicate');
   -- Expected: UNIQUE violation error (org_id + key already exists)
   ```

4. **Application Functionality:** Navigate through the dashboard:
   - /dashboard loads correctly
   - /clients shows the correct client list
   - /templates loads
   - /schedules loads
   - /settings shows correct email settings
   - /email-logs loads

5. **Cron Verification (optional):** If you want to test cron isolation, trigger the reminders cron manually via:
   ```
   curl -H "Authorization: Bearer <CRON_SECRET>" https://your-app.vercel.app/api/cron/reminders
   ```
   Check the response — it should show per-org processing results.
  </how-to-verify>
