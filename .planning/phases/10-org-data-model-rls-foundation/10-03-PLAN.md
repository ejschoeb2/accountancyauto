<objective>
Refactor both cron jobs and the email sender to iterate over organisations, scope every query by org_id, and use per-org Postmark credentials. This ensures no cross-tenant email delivery can occur.

Purpose: Cron jobs use the service_role client which bypasses RLS. Without explicit org_id filtering in application code, the crons would process all orgs' data together, potentially sending Org A's reminders to Org B's clients. This plan adds the application-level org isolation.

Output: Refactored cron routes, scheduler, queue builder, and email sender — all org-scoped.
</objective>

**Refactor `app/api/cron/reminders/route.ts`:**

Change the route to:
1. Fetch all active organisations: `SELECT * FROM organisations WHERE subscription_status IN ('active', 'trialing')`
2. Loop over each org sequentially
3. Call `processReminders(adminClient, org)` for each org
4. Aggregate results across all orgs
5. If any org fails, catch the error, log it with org details, continue to next org
6. Return aggregated results showing per-org breakdown

```typescript
// Pseudocode structure:
const { data: orgs } = await adminClient
  .from('organisations')
  .select('id, name, slug, postmark_server_token')
  .in('subscription_status', ['active', 'trialing']);

const allResults = [];
for (const org of orgs) {
  console.log(`Processing reminders for org: ${org.name} (${org.id})`);
  try {
    const result = await processReminders(adminClient, org);
    allResults.push({ org: org.name, ...result });
  } catch (error) {
    console.error(`Failed processing org ${org.name} (${org.id}):`, error);
    allResults.push({ org: org.name, error: error.message });
  }
}
```

**Refactor `app/api/cron/send-emails/route.ts`:**

Apply the same org iteration pattern as the reminders cron:

1. Fetch all active organisations (same query as reminders cron)
2. For each org:

   b. Query pending reminders with `.eq('org_id', org.id)`
   c. For each pending reminder, send email using org's Postmark token

async function getEmailFromForOrg(
  supabase: SupabaseClient,
  orgId: string
): Promise<{ from: string; replyTo: string }> {
  const { data } = await supabase
    .from('app_settings')
    .select('key, value')

}
