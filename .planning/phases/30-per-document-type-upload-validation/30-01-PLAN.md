**1. Create `lib/documents/validate.ts`**

Define and export:

```typescript
export interface ValidationWarning {
  code: string;       // e.g. 'BANK_STMT_DATE_MISMATCH', 'P60_YEAR_MISMATCH'
  message: string;    // Human-readable, actionable, referencing expected period
  expected?: string;  // Expected value (e.g. "2024-25")
  found?: string;     // Found value from OCR/parsing (may be null if not found)
}

export interface ValidationResult {
  warnings: ValidationWarning[];  // Empty array = no issues
}
```

Export `runValidation()` with this signature:

```typescript
export async function runValidation(
  documentTypeCode: string | null,
  mimeType: string,
  buffer: Buffer,
  portalTaxYear: string,
  extractedTaxYear: string | null,
  filingTypeId: string,
): Promise<ValidationResult>
```

**Implementation rules (all per CONTEXT.md locked decisions):**

- All per-type checks are **advisory warnings** — warn but accept. Never reject a document.
- Multiple warnings stack — return all that fire, not just the first.
- Warning messages must be specific and actionable, referencing the expected period.
- Only the top 5 document types get tailored checks: `BANK_STATEMENT`, `VAT_RETURN_WORKINGS`, `P60`, `P45`, `SA302`.
- For any other documentTypeCode (or null), return `{ warnings: [] }` immediately.

**Per-type check implementations:**

**P60 / P45 / SA302** (tax year mismatch):
- If `extractedTaxYear` is null, skip (no OCR data to compare).
- Normalise `portalTaxYear`: if it contains `-` (e.g. `"2024-25"`), extract the terminal year and expand 2-digit to 4-digit. If no `-`, use as-is.
- Compare normalised portal year with `extractedTaxYear`. On mismatch, return warning with code `{docTypeCode}_YEAR_MISMATCH`.
- Message: `"This {docTypeCode} appears to be for tax year {found} but we requested documents for {portalTaxYear}. Please check you've uploaded the right document."`

**BANK_STATEMENT** (two sub-checks):
- **Sub-check A: PDF date range** — Only for `mimeType === 'application/pdf'`.
  - Import and call `extractPdfText(buffer)` from `lib/documents/ocr.ts`. If it throws or returns `isImageOnly`, skip this sub-check.
  - Search the extracted text for period markers using targeted regexes. Look for patterns like:
    - `Statement Period: DD/MM/YYYY to DD/MM/YYYY`
    - `From: DD/MM/YYYY To: DD/MM/YYYY`
    - `Period: DD/MM/YYYY - DD/MM/YYYY`
  - **CRITICAL (Pitfall 3 from research):** Only look for structured period markers (Statement Period, From/To, Period), NOT any date in the document. If no period marker is found, skip entirely — do NOT warn on absence.
  - If a period is found, parse start and end dates. The portal `taxYear` means April {taxYear-1} to April {taxYear} for UK tax (or calendar year if it's just YYYY). Check if the statement period overlaps with the expected range. If no overlap, warn with code `BANK_STMT_PERIOD_MISMATCH`.
  - Message: `"This bank statement covers {found_start} to {found_end} which doesn't appear to fall within the {portalTaxYear} tax period. Please check you've uploaded the right statement."`

- **Sub-check B: Spreadsheet date column** — Only for spreadsheet MIME types (`application/vnd.ms-excel`, `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`, `text/csv`).
  - Import `xlsx` inside the function body (not module level — Pitfall 1).
  - Use `XLSX.read(buffer, { type: 'buffer', cellDates: true })`.
  - Get the first sheet, convert to JSON with `{ header: 1 }`.
  - Sample the first 20 rows. Check if any cell value is a `Date` instance (from `cellDates: true`), or alternatively check for date-like string patterns (DD/MM/YYYY, YYYY-MM-DD) in string cells.
  - If no date-like values found in any column, warn with code `BANK_STMT_NO_DATES`.
  - Message: `"This spreadsheet doesn't appear to contain any date columns. Bank statements usually include transaction dates — please check you've uploaded the correct file."`
  - Wrap entire SheetJS block in try/catch — on parse error, skip silently.

**VAT_RETURN_WORKINGS** (period plausibility):
- Only for `mimeType === 'application/pdf'`.
- Extract PDF text via `extractPdfText(buffer)`. If throws or isImageOnly, skip.
- Search for VAT period patterns: quarter date references like `DD/MM/YYYY`, period labels like "Quarter ending", "Period", "VAT period".
- **CRITICAL (Pitfall 4 from research):** Use a loose plausibility check — is the extracted period within plus or minus 1 year of `portalTaxYear`? Do NOT require exact stagger group alignment.
- If a VAT period is found and it falls outside +-1 year of the portal tax year, warn with code `VAT_PERIOD_IMPLAUSIBLE`.
- Message: `"This VAT return workings document references a period around {found_period} which seems outside the {portalTaxYear} tax year. Please check you've uploaded the right document."`
- If no VAT period markers found, skip — do not warn.

Create migration `add_validation_columns_to_client_documents` with SQL:

```sql
ALTER TABLE client_documents
  ADD COLUMN IF NOT EXISTS needs_review BOOLEAN NOT NULL DEFAULT false,
  ADD COLUMN IF NOT EXISTS validation_warnings JSONB DEFAULT NULL;

COMMENT ON COLUMN client_documents.needs_review IS 'True when per-type validation raised advisory warnings at upload time; cleared by accountant';
COMMENT ON COLUMN client_documents.validation_warnings IS 'Structured array of ValidationWarning objects from upload validation; null when no warnings';
```
