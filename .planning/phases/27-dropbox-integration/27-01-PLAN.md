    **Part A -- Create `lib/storage/dropbox.ts`:**

    This file implements the `StorageProvider` interface for Dropbox. Follow the rehydration pattern from 27-RESEARCH.md Pattern 1.

    **Constructor:**
    ```typescript
    export class DropboxProvider implements StorageProvider {
      private orgId: string;

      constructor(orgConfig: { id: string }) {
        this.orgId = orgConfig.id;
      }
    ```

    **Private helper -- `getAuthClient(): Promise<{ dbx: Dropbox; auth: DropboxAuth }>`:**
    1. Create admin client via `createAdminClient()`
    2. SELECT `dropbox_access_token_enc`, `dropbox_refresh_token_enc`, `dropbox_token_expires_at` from organisations WHERE id = this.orgId
    3. Throw `'Dropbox not connected for this organisation'` if `dropbox_refresh_token_enc` is null
    4. Construct `DropboxAuth` instance:
       ```typescript
       const auth = new DropboxAuth({
         clientId: process.env.DROPBOX_APP_KEY!,
         clientSecret: process.env.DROPBOX_APP_SECRET!,
         accessToken: org.dropbox_access_token_enc
           ? decryptToken(org.dropbox_access_token_enc)
           : undefined,
         refreshToken: decryptToken(org.dropbox_refresh_token_enc),
         accessTokenExpiresAt: org.dropbox_token_expires_at
           ? new Date(org.dropbox_token_expires_at)
           : undefined,
       });
       ```
    5. Call `await auth.checkAndRefreshAccessToken()` -- SDK handles refresh when needed
    6. After refresh, persist new access token and expiry back to Postgres:
       ```typescript
       const newAccessToken = auth.getAccessToken();
       const newExpiresAt = auth.getAccessTokenExpiresAt();
       if (newAccessToken) {
         await admin.from('organisations').update({
           dropbox_access_token_enc: encryptToken(newAccessToken),
           dropbox_token_expires_at: newExpiresAt?.toISOString() ?? null,
           storage_backend_status: 'active',
         }).eq('id', this.orgId);
       }
       ```
    7. Wrap the entire `checkAndRefreshAccessToken()` + persist block in a try/catch. On error: check for revocation indicators in the error message (e.g., 'invalid_grant', 'expired_access_token', 'Invalid refresh token'). On unrecoverable auth error:
       ```typescript
       await admin.from('organisations').update({
         storage_backend_status: 'reauth_required',
         dropbox_access_token_enc: null,
         dropbox_refresh_token_enc: null,
         dropbox_token_expires_at: null,
       }).eq('id', this.orgId);
       throw err; // re-throw -- never retry
       ```
    8. Return `{ dbx: new Dropbox({ auth }), auth }`

    **`upload(params: UploadParams): Promise<{ storagePath: string }>`:**
    1. Call `getAuthClient()` to get `{ dbx }`
    2. Build a path RELATIVE to app folder root (Dropbox prepends /Apps/Prompt/ automatically when app type = "App Folder"):
       ```typescript
       const ext = params.originalFilename.split('.').pop()?.toLowerCase() ?? 'bin';
       const uuid = crypto.randomUUID();
       const path = `/${params.clientId}/${params.filingTypeId}/${params.taxYear}/${uuid}.${ext}`;
       ```
       Do NOT prepend `/Apps/Prompt/` -- this would produce double-prefixed paths.
    3. Upload:
       ```typescript
       await dbx.filesUpload({
         path,
         contents: Buffer.from(params.file),
         mode: { '.tag': 'add' },
         autorename: false,
       });
       ```
    4. Return `{ storagePath: path }` -- the relative path is stored in client_documents.storage_path

    **`getDownloadUrl(storagePath: string): Promise<{ url: string }>`:**
    1. Call `getAuthClient()` to get `{ dbx }`
    2. Call `dbx.filesGetTemporaryLink({ path: storagePath })`
    3. Return `{ url: (response.result as any).link }` -- 4-hour TTL, directly downloadable

    **`getBytes(storagePath: string): Promise<Buffer>`:**
    1. Call `getDownloadUrl(storagePath)` to get the temporary link
    2. Fetch the URL: `const response = await fetch(url)`
    3. If not ok, throw `Dropbox getBytes failed: ${response.status}`
    4. Return `Buffer.from(await response.arrayBuffer())`

    **`delete(storagePath: string): Promise<void>`:**
    1. Call `getAuthClient()` to get `{ dbx }`
    2. Call `dbx.filesDeleteV2({ path: storagePath })`

    **Imports:**
    ```typescript
    import { Dropbox, DropboxAuth } from 'dropbox';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { encryptToken, decryptToken } from '@/lib/crypto/tokens';
    import type { StorageProvider, UploadParams } from '@/lib/documents/storage';
    ```

    **Critical anti-patterns (from research):**
    - Do NOT construct `Dropbox` directly with `{ accessToken }` -- this skips token refresh. Always use `new Dropbox({ auth })`.
    - Do NOT prefix paths with `/Apps/Prompt/` -- app folder access type handles this.
    - Do NOT read env vars at module level -- follow the lazy pattern from D-11-05-01 and D-24-03-01.
    - Do NOT store plaintext tokens -- every write to `_enc` columns goes through `encryptToken()`.

    **`DROPBOX_APP_SECRET`**

    - Format: Alphanumeric string

    - Environment: Production + Preview (different values per environment)
