    The route already branches on `doc.storage_backend`. The `resolveProvider()` factory already handles `case 'dropbox'`. For Dropbox documents, `provider.getBytes(storagePath)` calls `DropboxProvider.getBytes()` which fetches via `filesGetTemporaryLink` then downloads the bytes. No code change may be needed -- just verify the branching covers all backends (not just `google_drive` special case).

    If the existing code looks like:
    ```typescript
    if (doc.storage_backend === 'google_drive') {
      // provider.getBytes() path
    } else {
      // getSignedDownloadUrl() path
    }
    ```
    Then Dropbox documents fall through to the `else` branch which calls `getSignedDownloadUrl()` -- this would FAIL for Dropbox. Fix by changing the branching to:
    ```typescript
    if (!doc.storage_backend || doc.storage_backend === 'supabase') {
      // Supabase path: getSignedDownloadUrl() + fetch
      const { signedUrl } = await getSignedDownloadUrl(doc.storage_path);
      const response = await fetch(signedUrl);
      if (!response.ok) { console.warn(...); continue; }
      buffer = await response.arrayBuffer();
    } else {
      // All third-party backends (google_drive, dropbox): use resolveProvider().getBytes()
      const provider = resolveProvider({
        id: org?.id ?? '',
        storage_backend: (org?.storage_backend ?? 'supabase') as StorageBackend,
      });
      const bytes = await provider.getBytes(doc.storage_path);
      buffer = bytes.buffer.slice(bytes.byteOffset, bytes.byteOffset + bytes.byteLength);
    }
    ```
    This approach is forward-compatible -- any new backend added in the future also routes through resolveProvider.getBytes() automatically.

    **If Phase 25 NOT applied (single-path getSignedDownloadUrl only):**
    Follow the full Phase 25-05 DSAR update pattern:
    1. Add `storage_backend` to docs SELECT
    2. Fetch org storage config once before the loop
    3. Branch per document as shown above

    if (doc.storage_backend === 'google_drive') {
