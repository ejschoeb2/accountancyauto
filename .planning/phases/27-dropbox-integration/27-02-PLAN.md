<objective>
Build the two OAuth2 route handlers for Dropbox connect/callback, the disconnect server action, and the DropboxConnectCard component.

Purpose: The connect route generates the Dropbox authorization URL with token_access_type=offline and CSRF state protection. The callback route validates state, exchanges the code for tokens, validates refresh_token presence (DRPBX-01), and persists encrypted credentials. The disconnect action clears tokens and resets the backend. The connect card provides the Settings UI for these actions. Together these fulfill DRPBX-01 and DRPBX-05.
Output: app/api/auth/dropbox/connect/route.ts, app/api/auth/dropbox/callback/route.ts, updated app/actions/settings.ts, app/(dashboard)/settings/components/dropbox-connect-card.tsx.
</objective>

  <action>
    **A. Create `app/api/auth/dropbox/connect/route.ts`:**

    A single `GET` handler that:
    1. Verify the user is authenticated via `createClient()` and `supabase.auth.getUser()` -- return 401 JSON response if not authenticated
    2. Get `orgId` via `getOrgContext()`
    3. Generate a UUID CSRF state token: `crypto.randomUUID()`
    4. Store the state in organisations (DB-based CSRF):
       ```typescript
       const admin = createAdminClient();
       await admin.from('organisations')
         .update({ dropbox_oauth_state: state })
         .eq('id', orgId);
       ```
    5. Construct a `DropboxAuth` instance lazily (NOT at module level -- prevent build failure when env vars absent):
       ```typescript
       const auth = new DropboxAuth({
         clientId: process.env.DROPBOX_APP_KEY!,
       });
       ```
    6. Generate the authorization URL with `token_access_type=offline` as the CRITICAL 4th argument:
       ```typescript
       const authUrl = await auth.getAuthenticationUrl(
         process.env.DROPBOX_REDIRECT_URI!,
         state,
         'code',        // response_type
         'offline',     // token_access_type -- REQUIRED for refresh token (DRPBX-01)
       );
       ```
    7. Return `NextResponse.redirect(authUrl as string)`

    Imports:
    ```typescript
    import { NextResponse } from 'next/server';
    import { createClient } from '@/lib/supabase/server';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { getOrgContext } from '@/lib/auth/org-context';
    import { DropboxAuth } from 'dropbox';
    ```

    **B. Create `app/api/auth/dropbox/callback/route.ts`:**

    A single `GET` handler that:

    1. **Auth check:**
       - Verify user is authenticated via `createClient()` and `supabase.auth.getUser()` -- redirect to `/login` if not

    2. **CSRF state validation:**
       - Extract `code` and `state` from URL search params
       - Get `orgId` via `getOrgContext()`
       - Use admin client to look up the org's `dropbox_oauth_state`:
         ```typescript
         const admin = createAdminClient();
         const { data: org } = await admin.from('organisations')
           .select('dropbox_oauth_state')
           .eq('id', orgId)
           .single();
         ```
       - If `state` does not match `org?.dropbox_oauth_state` or either is missing: redirect to `/settings?tab=storage&error=invalid_state` (do NOT throw -- redirect is cleaner UX)

    3. **Token exchange:**
       - Construct `DropboxAuth` lazily:
         ```typescript
         const auth = new DropboxAuth({
           clientId: process.env.DROPBOX_APP_KEY!,
           clientSecret: process.env.DROPBOX_APP_SECRET!,
         });
         ```
       - Exchange code: `const tokenResponse = await auth.getAccessTokenFromCode(process.env.DROPBOX_REDIRECT_URI!, code!)`
       - Cast result: `const result = tokenResponse.result as any`

    4. **DRPBX-01: Mandatory refresh_token presence check:**
       ```typescript
       if (!result.refresh_token) {
         // token_access_type=offline was not in auth URL -- reject immediately
         return NextResponse.redirect(
           `${process.env.NEXT_PUBLIC_APP_URL}/settings?tab=storage&error=dropbox_no_refresh_token`
         );
       }
       ```
       Do NOT store the short-lived access token if refresh_token is missing.

    5. **Persist encrypted tokens:**
       ```typescript
       const accessToken = result.access_token;
       const refreshToken = result.refresh_token;
       const expiresIn = result.expires_in; // seconds from now
       const expiresAt = new Date(Date.now() + expiresIn * 1000);

       await admin.from('organisations').update({
         dropbox_refresh_token_enc: encryptToken(refreshToken),
         dropbox_access_token_enc: encryptToken(accessToken),
         dropbox_token_expires_at: expiresAt.toISOString(),
         storage_backend: 'dropbox',
         storage_backend_status: 'active',
         dropbox_oauth_state: null, // Clear CSRF token
       }).eq('id', orgId);
       ```

    6. **Redirect on success:**
       ```typescript
       return NextResponse.redirect(
         `${process.env.NEXT_PUBLIC_APP_URL}/settings?tab=storage&connected=dropbox`
       );
       ```

    7. Wrap the entire token exchange + DB write in a try/catch. On error: redirect to `/settings?tab=storage&error=dropbox_exchange_failed`

    Imports:
    ```typescript
    import { NextRequest, NextResponse } from 'next/server';
    import { createClient } from '@/lib/supabase/server';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { getOrgContext } from '@/lib/auth/org-context';
    import { DropboxAuth } from 'dropbox';
    import { encryptToken } from '@/lib/crypto/tokens';
    ```

    **Critical anti-patterns:**
    - Do NOT use a cookie for CSRF state -- use the DB column `dropbox_oauth_state` (already added in Plan 01 migration)
    - Do NOT skip the `result.refresh_token` presence check -- this is the core of DRPBX-01
    - Do NOT write plaintext tokens to any DB column -- every value goes through `encryptToken()`

    ```typescript
    export async function disconnectDropbox(): Promise<{ error?: string }> {
      const { orgId, orgRole } = await getOrgContext();
      if (orgRole !== 'admin') return { error: 'Admin only' };

      const admin = createAdminClient();
      const { error } = await admin.from('organisations').update({
        dropbox_refresh_token_enc: null,
        dropbox_access_token_enc: null,
        dropbox_token_expires_at: null,
        storage_backend: 'supabase',
        storage_backend_status: null,
      }).eq('id', orgId);

      if (error) return { error: error.message };
      revalidatePath('/settings');
      return {};
    }
    ```
