**settings.ts changes:**

1. Add new user-aware versions of getSendHour and getEmailSettings that accept an optional userId parameter and implement the fallback pattern:

```typescript
export async function getUserSendHour(): Promise<number> {
  const supabase = await createClient();
  const orgId = await getOrgId();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return 9;

  // Try user-specific row first
  const { data: userRow } = await supabase
    .from('app_settings')
    .select('value')
    .eq('org_id', orgId)
    .eq('user_id', user.id)
    .eq('key', 'reminder_send_hour')
    .maybeSingle();
  if (userRow) return parseInt(userRow.value, 10);

  // Fallback to org-level default (user_id IS NULL)
  const { data: orgRow } = await supabase
    .from('app_settings')
    .select('value')
    .eq('org_id', orgId)
    .is('user_id', null)
    .eq('key', 'reminder_send_hour')
    .maybeSingle();

  return orgRow ? parseInt(orgRow.value, 10) : 9;
}
```

2. Add `updateUserSendHour(hour: number)` — upserts with `{ org_id: orgId, user_id: userId, key: 'reminder_send_hour', value: String(hour) }` using `onConflict: 'org_id,user_id,key'`.

3. Add `getUserEmailSettings()` — same fallback pattern: try user-specific rows for email_sender_name, email_sender_address, email_reply_to, fall back to org defaults. Use `.is('user_id', null)` for the org fallback query (NOT `.eq('user_id', null)` — PostgREST requires `.is()` for IS NULL).

4. Add `updateUserEmailSettings(settings: EmailSettings)` — upserts three rows with user_id set, using `onConflict: 'org_id,user_id,key'`.

5. Update the existing admin `updateSendHour` and `updateEmailSettings` functions to include `user_id: null` in the upsert row AND update the conflict target from `onConflict: 'org_id,key'` to `onConflict: 'org_id,user_id,key'`. This is CRITICAL — the old conflict target will fail after the unique constraint migration (Pitfall 1 from research).

6. Similarly update `updateSetupMode`, `markOnboardingComplete`, `updateInboundCheckerMode` — these all use `onConflict: 'org_id,key'` and must change to `onConflict: 'org_id,user_id,key'` with `user_id: null` in the upsert payload. These are org-level settings (no per-user variant needed).

IMPORTANT: The `getSendHour` (existing, admin) and `getEmailSettings` (existing, admin) should continue to read org-level defaults (filter by `user_id IS NULL` explicitly now). The existing `updateSendHour` and `updateEmailSettings` are org-level admin actions.
