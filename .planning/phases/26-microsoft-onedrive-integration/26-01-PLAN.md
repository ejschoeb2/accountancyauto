<task type="auto">
  <name>Task 2: Create lib/storage/msal-cache-plugin.ts</name>
  <files>lib/storage/msal-cache-plugin.ts</files>
  <action>
    Create `lib/storage/msal-cache-plugin.ts`. This is the persistence bridge between MSAL's in-memory token cache and the encrypted `ms_token_cache_enc` column in organisations.

    The file must export:

    **`PostgresMsalCachePlugin` class implementing ICachePlugin:**

    ```typescript
    import type { ICachePlugin, TokenCacheContext } from '@azure/msal-node';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { encryptToken, decryptToken } from '@/lib/crypto/tokens';
    ```

    Constructor takes `orgId: string` (private readonly). One instance per org per request — never share across orgs.

    **`beforeCacheAccess(cacheContext: TokenCacheContext): Promise<void>`:**
    1. Create admin client via `createAdminClient()`
    2. SELECT `ms_token_cache_enc` from `organisations` WHERE `id = this.orgId`
    3. If `data?.ms_token_cache_enc` is truthy:
       - Decrypt: `const decrypted = decryptToken(data.ms_token_cache_enc)`
       - Deserialize into MSAL memory: `cacheContext.tokenCache.deserialize(decrypted)`
    4. If no cache exists yet (fresh connect), MSAL starts with an empty in-memory cache — this is correct and expected

    **`afterCacheAccess(cacheContext: TokenCacheContext): Promise<void>`:**
    1. Check `cacheContext.cacheHasChanged` — if false, return early (no DB write needed)
    2. Serialize MSAL cache: `const serialized = cacheContext.tokenCache.serialize()`
    3. Encrypt: `const encrypted = encryptToken(serialized)`
    4. UPDATE `organisations` SET `ms_token_cache_enc = encrypted` WHERE `id = this.orgId`

    Critical implementation notes:
    - Do NOT encrypt individual tokens within the MSAL cache — encrypt the entire serialized cache blob as a single unit (per research anti-pattern)
    - Do NOT read env vars at module level — only `createAdminClient()` is called at method invocation time (it already handles lazy init)
    - The class is stateless between method calls — each method creates its own admin client
  </action>
