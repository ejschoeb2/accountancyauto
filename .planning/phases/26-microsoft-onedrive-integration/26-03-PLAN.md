  <files>app/api/auth/onedrive/connect/route.ts, app/api/auth/onedrive/callback/route.ts</files>

    **A. Create `app/api/auth/onedrive/connect/route.ts`:**

    A single `GET` handler:

    1. Verify the user is authenticated via `createClient()` and `supabase.auth.getUser()` — return 401 if not authenticated.

    2. Create a `ConfidentialClientApplication` lazily inside the handler (NOT at module level — prevent build failure when env vars are absent at build time):
       ```typescript
       const msalClient = new ConfidentialClientApplication({
         auth: {
           clientId: process.env.MS_CLIENT_ID!,
           authority: 'https://login.microsoftonline.com/common',
           clientSecret: process.env.MS_CLIENT_SECRET!,
         },
       });
       ```
       No `cachePlugin` needed for the connect route — it only generates the auth URL, no tokens are involved yet.

    3. Generate a 32-byte hex CSRF state token: `crypto.randomBytes(32).toString('hex')`

    4. Generate the authorization URL:
       ```typescript
       const authUrl = await msalClient.getAuthCodeUrl({
         scopes: ['Files.ReadWrite', 'offline_access'],
         redirectUri: process.env.MS_REDIRECT_URI!,
         state,
         prompt: 'consent', // CRITICAL: ensures refresh token in cache; forces account picker
       });
       ```

    5. Create a `NextResponse.redirect(authUrl)` response

    6. Set the state cookie:
       ```typescript
       response.cookies.set('ms_oauth_state', state, {
         httpOnly: true,
         secure: process.env.NODE_ENV === 'production',
         sameSite: 'lax',
         maxAge: 600, // 10 minutes
         path: '/',
       });
       ```

    7. Return the redirect response

    Imports:
    ```typescript
    import { NextResponse } from 'next/server';
    import { createClient } from '@/lib/supabase/server';
    import { ConfidentialClientApplication } from '@azure/msal-node';
    import crypto from 'crypto';
    ```

    **B. Create `app/api/auth/onedrive/callback/route.ts`:**

    A single `GET` handler:

    1. **Auth + CSRF validation:**
       - Verify user is authenticated via `createClient()` and `supabase.auth.getUser()` — redirect to `/login` if not
       - Get `orgId` via `getOrgContext()`
       - Extract `code` and `state` from URL search params
       - Extract stored state from `request.cookies.get('ms_oauth_state')?.value`
       - If state mismatch or missing: redirect to `/settings?tab=storage&error=invalid_state` (do NOT throw)

    2. **Token exchange with MSAL:**
       - Create `ConfidentialClientApplication` WITH the PostgresMsalCachePlugin (orgId is known):
         ```typescript
         const msalClient = new ConfidentialClientApplication({
           auth: {
             clientId: process.env.MS_CLIENT_ID!,
             authority: 'https://login.microsoftonline.com/common',
             clientSecret: process.env.MS_CLIENT_SECRET!,
           },
           cache: {
             cachePlugin: new PostgresMsalCachePlugin(orgId),
           },
         });
         ```
         CRITICAL: The cachePlugin MUST be present so afterCacheAccess fires after acquireTokenByCode, persisting the initial token cache to Postgres.

       - Exchange code for tokens:
         ```typescript
         const tokenResponse = await msalClient.acquireTokenByCode({
           code: code!,
           redirectUri: process.env.MS_REDIRECT_URI!,
           scopes: ['Files.ReadWrite', 'offline_access'],
         });
         ```
         afterCacheAccess fires automatically — the encrypted cache is now persisted to ms_token_cache_enc.

    3. **AADSTS53003 handling:**
       Wrap the acquireTokenByCode call in try/catch:
       ```typescript
       try {
         const tokenResponse = await msalClient.acquireTokenByCode({ ... });
         // success path below
       } catch (error: unknown) {
         const message = error instanceof Error ? error.message : String(error);
         if (message.includes('AADSTS53003')) {
           const errorUrl = new URL('/settings', request.url);
           errorUrl.searchParams.set('tab', 'storage');
           errorUrl.searchParams.set('error', 'conditional_access_blocked');
           const errorResponse = NextResponse.redirect(errorUrl);
           errorResponse.cookies.delete('ms_oauth_state');
           return errorResponse;
         }
         console.error('OneDrive OAuth callback error:', error);
         const errorUrl = new URL('/settings', request.url);
         errorUrl.searchParams.set('tab', 'storage');
         errorUrl.searchParams.set('error', 'auth_failed');
         const errorResponse = NextResponse.redirect(errorUrl);
         errorResponse.cookies.delete('ms_oauth_state');
         return errorResponse;
       }
       ```

    4. **Persist homeAccountId and update org storage backend:**
       - Extract `homeAccountId` from `tokenResponse.account!.homeAccountId`
       - Use `createAdminClient()` to UPDATE organisations:
         ```typescript
         await admin.from('organisations').update({
           storage_backend: 'onedrive',
           storage_backend_status: 'active',
           ms_home_account_id: tokenResponse.account!.homeAccountId,
           // ms_token_cache_enc is already persisted by afterCacheAccess
         }).eq('id', orgId);
         ```
       - Wrap in try/catch: on DB error, redirect to `/settings?tab=storage&error=db_error`

    5. **Clean up and redirect:**
       - Build success redirect URL: `/settings?tab=storage&connected=onedrive`
       - Create `NextResponse.redirect(successUrl)`
       - Delete the state cookie: `response.cookies.delete('ms_oauth_state')`
       - Return the redirect

    All errors redirect to `/settings` with an error query param — OAuth callbacks must never show raw error pages.

    Imports:
    ```typescript
    import { NextRequest, NextResponse } from 'next/server';
    import { createClient } from '@/lib/supabase/server';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { getOrgContext } from '@/lib/auth/org-context';
    import { ConfidentialClientApplication } from '@azure/msal-node';
    import { PostgresMsalCachePlugin } from '@/lib/storage/msal-cache-plugin';
    ```
