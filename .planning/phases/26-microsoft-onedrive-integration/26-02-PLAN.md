  <action>
    Create `lib/storage/onedrive.ts`. This file implements the `StorageProvider` interface for Microsoft OneDrive using raw `fetch` against the Microsoft Graph REST API. MSAL handles all token refresh internally via the PostgresMsalCachePlugin.

    **Constructor:**
    ```typescript
    export class OneDriveProvider implements StorageProvider {
      constructor(
        private readonly orgId: string,
        private readonly homeAccountId: string
      ) {}
    ```
    The `orgId` is used to create the MSAL client with the org-specific cache plugin. The `homeAccountId` is used for `getAccountByHomeId()` to look up the correct account in the cache.

    **Private helper — `createMsalClient(): ConfidentialClientApplication`:**
    Creates a `ConfidentialClientApplication` with:
    - `auth.clientId`: `process.env.MS_CLIENT_ID!`
    - `auth.authority`: `'https://login.microsoftonline.com/common'`
    - `auth.clientSecret`: `process.env.MS_CLIENT_SECRET!`
    - `cache.cachePlugin`: `new PostgresMsalCachePlugin(this.orgId)`

    CRITICAL: Do NOT call this at module level or store at instance level across requests. Each method call creates a fresh MSAL client because the `PostgresMsalCachePlugin` fires `beforeCacheAccess` on the first cache operation within that client instance.

    **Private helper — `getAccessToken(): Promise<string>`:**
    1. Create MSAL client via `this.createMsalClient()`
    2. Get token cache: `const tokenCache = msalClient.getTokenCache()`
    3. Look up account: `const account = await tokenCache.getAccountByHomeId(this.homeAccountId)`
    4. If `account === null`:
       - Set `storage_backend_status = 'reauth_required'` in organisations via admin client
       - Clear `ms_token_cache_enc = null` and `ms_home_account_id = null`
       - Throw: `'OneDrive session not found — reconnection required'`
    5. Call `acquireTokenSilent({ account, scopes: ['Files.ReadWrite', 'offline_access'] })`
    6. Wrap in try/catch:
       - On `InteractionRequiredAuthError` (import from `@azure/msal-node`):
         - UPDATE organisations SET `storage_backend_status = 'reauth_required'`, `ms_token_cache_enc = null`, `ms_home_account_id = null` WHERE id = this.orgId
         - Re-throw — never retry
       - On any other error: re-throw as-is
    7. Return `result.accessToken`

    afterCacheAccess fires automatically inside acquireTokenSilent if MSAL refreshes the token — no manual persist needed.

    **`upload(params: UploadParams): Promise<{ storagePath: string }>`:**
    1. Get access token via `this.getAccessToken()`
    2. Build the path segments, URL-encoding each segment individually:
       ```typescript
       const clientName = params.clientName ?? params.clientId;
       const encodedPath = [
         encodeURIComponent(clientName),
         encodeURIComponent(params.filingTypeId),
         encodeURIComponent(params.taxYear),
         encodeURIComponent(params.originalFilename),
       ].join('/');
       ```
    3. PUT to Graph API (simple upload for files ≤ 4 MB — Phase 29 handles chunked upload for larger files):
       ```typescript
       const url = `https://graph.microsoft.com/v1.0/me/drive/root:/Apps/Prompt/${encodedPath}:/content`;
       const response = await fetch(url, {
         method: 'PUT',
         headers: {
           Authorization: `Bearer ${accessToken}`,
           'Content-Type': params.mimeType,
         },
         body: Buffer.from(params.file),
       });
       ```
    4. Check `response.ok` — throw with status and body text on failure
    5. Parse response JSON: `const data = await response.json()`
    6. Return `{ storagePath: data.id }` — the OneDrive item ID is the storagePath

    Key advantage: No folder creation helper needed. OneDrive's path-based PUT auto-creates intermediate folders.

    **`getDownloadUrl(storagePath: string): Promise<{ url: string }>`:**
    Unlike Google Drive (which throws here), OneDrive provides `@microsoft.graph.downloadUrl`:
    1. Get access token via `this.getAccessToken()`
    2. Fetch item metadata:
       ```typescript
       const url = `https://graph.microsoft.com/v1.0/me/drive/items/${storagePath}?$select=id,%40microsoft.graph.downloadUrl`;
       const response = await fetch(url, {
         headers: { Authorization: `Bearer ${accessToken}` },
       });
       ```
    3. Parse JSON and extract `data['@microsoft.graph.downloadUrl']`
    4. If downloadUrl is falsy, throw `'OneDrive downloadUrl not returned — item may not be accessible'`
    5. Return `{ url: downloadUrl }`

    Note: This URL is pre-authenticated (no Authorization header needed) but expires within minutes — always fetch fresh.

    **`getBytes(storagePath: string): Promise<Buffer>`:**
    1. Get access token via `this.getAccessToken()`
    2. Fetch file content:
       ```typescript
       const url = `https://graph.microsoft.com/v1.0/me/drive/items/${storagePath}/content`;
       const response = await fetch(url, {
         headers: { Authorization: `Bearer ${accessToken}` },
       });
       ```
    3. Check `response.ok` — throw on failure
    4. Convert to Buffer: `Buffer.from(await response.arrayBuffer())`

    **`delete(storagePath: string): Promise<void>`:**
    1. Get access token via `this.getAccessToken()`
    2. DELETE the item:
       ```typescript
       const url = `https://graph.microsoft.com/v1.0/me/drive/items/${storagePath}`;
       const response = await fetch(url, {
         method: 'DELETE',
         headers: { Authorization: `Bearer ${accessToken}` },
       });
       ```
    3. OneDrive returns 204 No Content on success. Check `response.ok` — throw on failure.

    Imports:
    ```typescript

      ConfidentialClientApplication,
      InteractionRequiredAuthError,
    } from '@azure/msal-node';
    import { createAdminClient } from '@/lib/supabase/admin';
    import { PostgresMsalCachePlugin } from './msal-cache-plugin';
    import type { StorageProvider, UploadParams } from '@/lib/documents/storage';
    ```
