**lib/reminders/queue-builder.ts:**

In `buildReminderQueue`, after fetching `client_filing_assignments` and before the main loop that iterates over assignments, add:

1. Query `org_filing_type_selections` for the current org where `is_active = true`:
   ```typescript
   const { data: orgActiveTypes } = await supabase
     .from('org_filing_type_selections')
     .select('filing_type_id')
     .eq('org_id', org.id)
     .eq('is_active', true);
   const activeTypeIds = new Set((orgActiveTypes ?? []).map(r => r.filing_type_id));
   ```

2. At the start of the assignment loop (inside the `for` that iterates client filing assignments), add a guard:
   ```typescript
   if (!activeTypeIds.has(assignment.filing_type_id)) { skipped++; continue; }
   ```

   This ONLY filters at queue-build time, NOT at send time (per research anti-pattern guidance). Entries already in the queue were scheduled when the type was active.

3. Also update `rebuildQueueForClient` with the same pattern — fetch org active types using the client's org_id, skip inactive types.

**lib/onboarding/seed-defaults.ts:**

3. Use `.upsert()` with `onConflict: 'org_id,filing_type_id'` for idempotency

This ensures every new org created through the wizard or any other flow gets the 5 default filing types activated.
