  <action>
Create `app/(auth)/setup/wizard/page.tsx` as a "use client" component.

Read `app/(auth)/onboarding/page.tsx` first to understand the WizardStepper usage pattern.

STRUCTURE:
```typescript
"use client";

import { useState, useTransition } from "react";
import { WizardStepper } from "@/components/wizard-stepper";
import { CsvImportStep } from "./components/csv-import-step";
import { ConfigStep } from "./components/config-step";
import { markMemberSetupComplete, getUserSendHour, getUserEmailSettings, getInboundCheckerMode } from "@/app/actions/settings";
import type { EmailSettings, InboundCheckerMode } from "@/app/actions/settings";

const STEPS = [
  { label: "Import Clients" },
  { label: "Configuration" },
];
```

STATE:
- `step: number` — 0 or 1 (0-indexed for WizardStepper)
- `sendHour: number | null` — null until fetched
- `emailSettings: EmailSettings | null`
- `inboundMode: InboundCheckerMode | null`
- `isCompleting: boolean` — true while markMemberSetupComplete is in-flight

PREFETCH DEFAULTS FOR STEP 2:
On mount (useEffect with [] dependency), call `getUserSendHour()`, `getUserEmailSettings()`, `getInboundCheckerMode()` in parallel with Promise.all. Store results in state. This ensures ConfigStep has defaults pre-loaded when the user reaches Step 2.

STEP 1 COMPLETION (`handleStep1Complete`):
- Set `step = 1` — advance to Step 2

STEP 2 COMPLETION (`handleStep2Complete`):
- Set `isCompleting = true`
- Call `await markMemberSetupComplete()`
- If error: show an error state (simple text) and do NOT redirect
- If success: redirect to `/` using `window.location.href = "/"` in dev or the org subdomain root in prod. For simplicity, use `window.location.href = "/"` — the middleware handles subdomain resolution and will redirect to the correct org dashboard.

RENDER:
```tsx
<div className="space-y-8">
  <WizardStepper steps={STEPS} currentStep={step} />

  {step === 0 && (
    <CsvImportStep onComplete={handleStep1Complete} />
  )}

  {step === 1 && sendHour !== null && emailSettings !== null && inboundMode !== null && (
    <ConfigStep
      defaultSendHour={sendHour}
      defaultEmailSettings={emailSettings}
      defaultInboundMode={inboundMode}
      onComplete={handleStep2Complete}
    />
  )}

  {step === 1 && (sendHour === null || emailSettings === null || inboundMode === null) && (
    <div className="flex items-center justify-center py-12">
      <Loader2 className="size-8 animate-spin text-muted-foreground" />
    </div>
  )}
</div>
```

Import `Loader2` from "lucide-react" for the loading state.

ANTI-PATTERNS:
- Do NOT allow backward navigation from Step 2 to Step 1 — no "Back" button on Step 2
- Do NOT call markMemberSetupComplete in ConfigStep — it is called here after onComplete fires
- Do NOT use useRouter — use window.location.href for the final redirect (router.push doesn't force a full session reload)
