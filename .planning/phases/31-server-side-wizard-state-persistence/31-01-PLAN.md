**Server actions** — add to `app/(auth)/setup/wizard/actions.ts`:

1. Define `SetupDraft` interface at top of file:
```typescript
export interface SetupDraft {
  step: string;           // AdminStep value
  firmName?: string;
  firmSlug?: string;
  selectedTier?: string;  // PlanTier value
  importRows?: unknown[]; // EditableRow[] stored as JSON
  emailSubStep?: string;  // EmailSubStep value
  portalEnabled?: boolean;
  uploadCheckMode?: string;
  sendHour?: number;
  updatedAt: string;      // ISO timestamp
}
```
All fields optional except `step` and `updatedAt`.

2. `getSetupDraft()`:
- Follow the EXACT same pattern as `markOrgSetupComplete`: `createClient()` → `getUser()` → admin `user_organisations` lookup → admin `organisations` SELECT
- Returns `SetupDraft | null`
- Query: `admin.from("organisations").select("setup_draft").eq("id", membership.org_id).single()`
- Return `data?.setup_draft ?? null`

3. `saveSetupDraft(draft: SetupDraft)`:
- Same auth pattern as above
- Always set `draft.updatedAt = new Date().toISOString()` before saving
- Query: `admin.from("organisations").update({ setup_draft: draft }).eq("id", membership.org_id)`
- Returns `{ error?: string }`

4. Update `markOrgSetupComplete()`:
- Change the update payload from `{ setup_complete: true }` to `{ setup_complete: true, setup_draft: null }`
- This clears the draft when setup finishes (Pitfall 4: prevents race between fire-and-forget save and completion)

**CRITICAL anti-patterns to avoid:**
- Do NOT use `getOrgId()` from `lib/auth/org-context.ts` — it throws when no org exists. The wizard actions use the `user_organisations` lookup pattern which returns null gracefully.
- Do NOT save `wizard_pending_email` state — that stays in sessionStorage (set before user has account).

function hydrateFromDraft(draft: SetupDraft | null, overrideStep?: AdminStep) {
  if (!draft) return;
  setAdminStep(overrideStep ?? (draft.step as AdminStep));
  if (draft.firmName) setFirmName(draft.firmName);
  if (draft.firmSlug) setSlug(draft.firmSlug);
  if (draft.selectedTier) setSelectedTier(draft.selectedTier as PlanTier);

  if (draft.portalEnabled !== undefined) setClientPortalEnabled(draft.portalEnabled);

  if (draft.sendHour !== undefined) setSendHour(draft.sendHour);
  if (draft.emailSubStep) setEmailInitialSubStep(draft.emailSubStep as EmailSubStep);
  setOrgCreated(true);
}

**Step 5: Add draft save on step transitions**

Find every place where `setAdminStep(nextStep)` is called for the admin wizard and add a fire-and-forget `saveSetupDraft` call BEFORE it — but ONLY when `orgCreated` is true (Pitfall 1: can't save before org exists).

Do NOT save when advancing to "complete" step to avoid Pitfall 4 (race with markOrgSetupComplete).

Pattern:
```typescript
// Before each setAdminStep(nextStep) call:
if (orgCreated && nextStep !== "complete") {
  const draft = collectCurrentState();
  draft.step = nextStep;
  saveSetupDraft(draft); // fire-and-forget — no await
}
setAdminStep(nextStep);
```

There are multiple places where step transitions happen (Next buttons, Back buttons, etc.). The simplest approach: create a wrapper:
```typescript
function advanceToStep(nextStep: AdminStep) {
  if (orgCreated && nextStep !== "complete") {
    const draft = collectCurrentState();
    draft.step = nextStep;
    saveSetupDraft(draft).catch((e) => console.warn("Draft save failed:", e));
  }
  setAdminStep(nextStep);
}
```
Then replace all `setAdminStep(x)` calls that represent user-initiated step transitions with `advanceToStep(x)`. Keep `setAdminStep` for the mount effect hydration (those are restoring state, not user transitions).
