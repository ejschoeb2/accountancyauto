**Migration file** (`supabase/migrations/20260309000002_create_setup_draft_clients.sql`):
```sql
-- Phase 31: Setup draft clients staging table for large CSV imports
CREATE TABLE setup_draft_clients (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid NOT NULL REFERENCES organisations(id) ON DELETE CASCADE,
  row_index integer NOT NULL,
  data jsonb NOT NULL,
  created_at timestamptz DEFAULT now(),
  UNIQUE (org_id, row_index)
);

ALTER TABLE setup_draft_clients ENABLE ROW LEVEL SECURITY;

-- Service role full access (all access goes through admin client in server actions)
CREATE POLICY "Service role full access to setup_draft_clients"
  ON setup_draft_clients FOR ALL TO service_role
  USING (true) WITH CHECK (true);
```

No authenticated policy needed — all access goes through admin client in server actions.

**Server actions** — add to `app/(auth)/setup/wizard/actions.ts`:

1. `saveDraftClients(rows: EditableRow[])`:
- Same auth pattern: `createClient()` → `getUser()` → admin `user_organisations` lookup
- First DELETE all existing staging rows for this org: `admin.from("setup_draft_clients").delete().eq("org_id", orgId)`
- Then bulk INSERT: `admin.from("setup_draft_clients").insert(rows.map((row, i) => ({ org_id: orgId, row_index: i, data: row })))`
- Returns `{ error?: string }`
- Import `EditableRow` type from `./components/csv-import-step`

2. `getDraftClients()`:
- Same auth pattern
- Query: `admin.from("setup_draft_clients").select("row_index, data").eq("org_id", orgId).order("row_index")`
- Returns `EditableRow[] | null` (null if no rows found)
- Map results: `data?.map(r => r.data as EditableRow) ?? null`
- Return null (not empty array) when no rows exist — null means "no draft import data"

3. `clearDraftClients()`:
- Same auth pattern
- Query: `admin.from("setup_draft_clients").delete().eq("org_id", orgId)`
- Returns `{ error?: string }`

4. Update `markOrgSetupComplete()` to also clear staging rows:
- After the existing `organisations.update()`, add: `await admin.from("setup_draft_clients").delete().eq("id", membership.org_id)`
- Wait: the FK column is `org_id` not `id` — use `.eq("org_id", membership.org_id)`
- This is a belt-and-suspenders cleanup alongside ON DELETE CASCADE

**Also:** Remove `importRows` from the `SetupDraft` interface (it now lives in the staging table, not JSONB). Update `saveSetupDraft` to not include `importRows`. Update `collectCurrentState` in page.tsx in Task 2.
